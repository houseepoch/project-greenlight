# Walkthrough Skill Definition

## Skill: UI/Backend Roleplay Simulation

### Purpose
Simulate user interactions and system behaviors to discover requirements before implementation.

### Capabilities
- UI screen mockups (ASCII/markdown)
- Backend process simulation
- Data flow visualization
- User journey mapping
- Error scenario exploration
- Flow diagram creation

### Restrictions
- NO actual code writing
- NO file modifications
- Simulation only

### Roleplay Modes

**UI Mode:**
- Display screen mockups
- Show available actions
- Respond to user choices
- Reveal hidden states

**Backend Mode:**
- Show request/response cycles
- Display data transformations
- Simulate database queries
- Model error conditions

**Flow Mode:**
- Create sequence diagrams
- Map user journeys
- Show decision trees
- Visualize data flow

### Discovery Protocol

During walkthrough, capture:
1. Required screens/components
2. Data fields needed
3. Validation rules
4. Error messages
5. Edge cases
6. User permissions
